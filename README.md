# 梦女文生成器 · mengnv-story.skill

> 一个专为追星女设计的沉浸式梦女文创作技能。输入偶像名字、场景、感觉，即刻生成一篇专属于你的第一人称情感短文。

---

## 这是什么

**梦女文**，是以偶像为男主角、以读者自己为女主角的第一人称沉浸式情感短文。这个 Skill 将梦女文的写作方法论封装为可复用的 AI 技能，让 AI 按照专业的写作结构和约束来生成，而不是随意发挥。

核心体验：**「快」和「当下」**——你脑海里刚冒出那个念头「假如此刻我和他在一起……」，打开即用，即刻满足。

---

## 核心能力

**五段式写作结构**
```
场景渲染 → 关系背景 → 触发事件 → 情感高点 → 意犹未尽收尾
```

**6种情感类型**

| 类型 | 核心基调 |
|---|---|
| 甜甜的 | 双向明牌的偏爱，直接坦荡 |
| 有点虐 | 克制的遗憾，身份差的无力感 |
| 暧昧未明 | 双向暗流，欲言又止 |
| 暗黑带感 | 温柔外壳下的偏执掌控 |
| 治愈型 | 陪伴接纳，被看见被允许脆弱 |
| 宿命型 | 跨越时间的重逢，双向惦念 |

**13条核心约束**（保证质量的底线）
- 零标签原则：性格全靠动作落地，不直接贴标签
- 开篇不出现第一人称，先渲染场景再引出人物
- 偶像在剧情中自然叫出女主真名 1—2 次
- 结尾不升华价值观，停在情绪最饱满的那一帧
- 偶像真实信息必须严谨，不确定的宁可模糊处理
- ……（完整约束见 `SKILL.md`）

## 使用方式

### 在扣子（Coze）中使用

1. 下载 `mengnv-story.skill` 文件
2. 打开 [coze.cn](https://coze.cn) → 扣子编程
3. 在技能页面选择「上传文件」，传入 `.skill` 文件
4. 部署后，在对话中输入 `@梦女文生成器` 并填写信息即可

### 输入格式

```
他：[偶像名字]
近期动态：[一句话，偶像真实近期动态，选填]
我：[你的名字]
性格标签：[例：话不多但细心，选填]
此刻的我：[你的身份或正在做的事，选填]
场景：[你想要的场景描述]
感觉：[甜甜的 / 有点虐 / 暧昧未明 / 暗黑带感 / 治愈型 / 宿命型]
```

### 输出示例

```
输入：
他：张真源
我：梁莹
场景：深夜演出后台，偶然遇见
感觉：暧昧未明

输出：
一篇 1500 字左右的沉浸式短文，
第一段全是场景渲染，
他在第二段出现，
「梁莹」两个字在情绪最高点被叫出来……
```

## 设计说明

这个 Skill 的方法论来自对大量梦女文样本的分析与实测，核心设计原则：

**样本是风格老师，约束是底线守卫**
不用完整样本注入（避免模型临摹情节），只用约束条款来定义边界，让模型在对的方向上自由发挥。

**零标签原则**
所有人物性格、气质全部拆解为可感知的动作和神态，禁止形容词直接概括，这是文字质感的核心来源。

**结尾必须有呼吸空间**
不做「动作定格」式截断，用环境呼应或人物微反应作为缓冲，让「悬而未决的余韵」真正悬在那里。

## 注意事项

- 涉及真实偶像姓名的内容由用户自行输入，平台不预设数据库
- 生成内容为 AI 虚构，与真实人物无关
- 暗黑带感类型有专项尺度限制，不涉及肢体强制、威胁或性暗示
